Key Type Description Examples
log_transactions boolean Should we log the transactions made by players
product_lore_footer.enabled boolean Should we add a nice footer to the lore of every product?
product_lore_footer.content List Content of the footer
transaction_item PreparedItem The item to be displayed for each transaction in the transaction menu
default_product_icon.empty PreparedItem not-null Default icon for empty products
A product is considered as 'empty' when it has nothing to benefit the player
in other words: no exp points, no items are given, no perks and no boosters.
default_product_icon.package PreparedItem not-null Default icon for packages
A package is a product that gives player booster, perks, exp or items
(for items, requires the amount of two or more)
To change the package name, you can set the option `package_name` that
can be seen more in the Product schema.
A product with one item only will not be considered as a package, the default
icon for that product is the item. However, you can change that by looking at
the below option ('default_product_icon.single_item_package')
default_product_icon.package_details PackageDetails not-null Details for the default icon for packages
default_product_icon.single_item_package boolean Should we treat single item as 'package'
As explained above, if a product only gives
the player one item, the default icon will be
that item. Set this option to true will make
it rendered as a package.
